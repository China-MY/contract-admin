## 一、系统概述

### 1.1 系统定位

面向中小企业的**双向合同（应收/应付）财务管理平台**，覆盖合同全生命周期管理（签订→执行→收款/付款→开票→归档），集成了项目、发票、资金、客户/供应商等关联业务，支持项目级利润核算。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 双向合同管理 | 应收合同（向客户收款）+ 应付合同（向供应商付款），统一数据模型 |
| 资金跟踪链 | 合同→收款/付款计划→收款/付款记录，三级层层汇总 |
| 票税一体化 | 销项/进项发票管理，含税/不含税自动计算，支持多税率 |
| 项目利润核算 | 基于合同维度的收入-支出利润分析，含利润率 |
| 到期提醒引擎 | 合同到期、收款/付款计划到期自动提醒，显示过期天数 |
| RBAC 权限体系 | 用户-角色-菜单权限，支持部门和岗位 |
| 可配置字典 | 合同类型、项目类型、支付方式、税率等11类字典可自定义 |

### 1.3 核心数据流

```
客户/供应商注册
      │
      ▼
  项目立项 ──→ 签订合同 ──→ 开票
      │            │           │
      │            ▼           ▼
      │        收款/付款计划    发票管理
      │            │
      │            ▼
      │        收款/付款记录
      │
      ▼
  项目收支统计
  利润核算
```

**索引建议**：
- 唯一索引：`(contract_no, direction)`
- 普通索引：`direction`, `status`, `receipt_status`, `payment_status`, `project_id`, `counterparty_id`

**字段计算公式**：
```
unreceived_amount = settlement_amount - received_amount
uninvoiced_amount = settlement_amount - invoiced_amount
receipt_progress = (received_amount / settlement_amount) × 100%
invoice_progress = (invoiced_amount / settlement_amount) × 100%

(应付侧同理)

profit = settlement_amount - procurement_settlement_amount
profit_margin = (profit / settlement_amount) × 100%
```

## 六、核心业务逻辑

### 6.1 资金跟踪链（三级联动）

```
合同层（Contract）
  ├── contract_amount / settlement_amount    ← 基础数据
  ├── received_amount / paid_amount          ← 由记录汇总更新
  └── receipt_status / payment_status        ← 自动计算
               ↑
计划层（PaymentPlan）
  ├── planned_amount                          ← 分期计划
  ├── paid_amount                             ← 由记录汇总更新
  └── status / progress                       ← 自动计算
               ↑
记录层（PaymentRecord）
  ├── amount                                  ← 实际收/付款
  ├── status (pending → confirmed)            ← 确认
  └── method / account / expense_type         ← 凭证信息
```

**更新逻辑**（以收款为例）：

```
1. 新建收款记录（status='pending'）
2. 确认收款（status → 'confirmed'）
3. 触发更新：
   a. 汇总该合同下所有已确认收款记录 → 更新 contract.received_amount
   b. 汇总该计划下所有已确认收款记录 → 更新 plan.paid_amount
   c. 重新计算：
      - contract.unreceived_amount = settlement_amount - received_amount
      - contract.receipt_progress = received_amount / settlement_amount × 100%
      - contract.receipt_status:
          received_amount = 0         → 'unreceived'
          0 < received < settlement   → 'partial'
          received >= settlement      → 'completed'
      - plan 层同样逻辑
```

### 6.2 票链跟踪逻辑

```
合同层
  ├── invoiced_amount / received_invoice_amount  ← 由发票表汇总
  ├── received_no_invoice    = 已收款 - 已开票
  ├── invoiced_no_receipt    = 已开票 - 已收款
  └── invoice_progress / invoice_receipt_progress
               ↑
发票层（Invoice）
  ├── direction = 'output' → 汇总到合同.invoiced_amount
  ├── direction = 'input'  → 汇总到合同.received_invoice_amount
  └── 计算不含税金额和税额
```

### 6.3 利润计算

```
# 应收合同侧
利润 = 结算金额 - 关联采购合同的结算金额
利润率 = 利润 / 结算金额 × 100%

# 项目统计侧（项目级利润）
项目收入 = SUM(关联应收合同.结算金额)
项目支出 = SUM(关联应付合同.结算金额)
项目利润 = 项目收入 - 项目支出
项目利润率 = 项目利润 / 项目收入 × 100%
```

### 6.4 到期提醒引擎

**数据来源**：

| 提醒类型 | 日期字段 | 关联 | 显示内容 |
|---------|---------|------|---------|
| 应收合同到期 | contract.end_date | contract.direction='receivable' | 合同编号+名称+剩余/过期天数 |
| 应付合同到期 | contract.end_date | contract.direction='payable' | 同上 |
| 收款计划到期 | plan.planned_date | plan.direction='receipt' | 合同名称+计划金额+日期 |
| 付款计划到期 | plan.planned_date | plan.direction='payment' | 同上 |

**计算逻辑**：
```
剩余天数 = planned_date/end_date - 今天日期      // 正数：还有X天到期
过期天数 = 今天日期 - planned_date/end_date       // 正数：已过期X天
排序策略：按过期天数降序排列（最紧迫的排最前）
过滤条件：状态不为 completed/paid 的记录
```
